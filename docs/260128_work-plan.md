# 작업 계획서: 데스크탑 REST 클라이언트 (260128)

## 1) 목적
- `docs/pre-project.md`를 기준으로 MVP 개발을 위한 작업 계획과 완료 검증 방법을 정리한다.
- 현재 단계는 **문서 작업만** 수행한다.

## 2) 기준 문서
- `docs/pre-project.md`

## 3) 공통 완료 기준
- 각 작업 항목은 “완료 확인 방법”을 통해 **수동 검증 가능**해야 한다.
- 기능 구현 전 단계에서는 설계/문서 산출물이 준비되어 있어야 한다.

## 4) 마일스톤별 작업 계획 및 완료 확인

### M1: UI 골격
**작업 항목**
- 메인 윈도우 레이아웃(좌/중/하) 정의 및 위젯 구조 설계
- 탭 기반 멀티 요청 편집 구조 설계
- Environment 선택/관리 UI 구상

**산출물**
- UI 레이아웃 와이어프레임 또는 화면 구성 문서
- 위젯 간 데이터 흐름 설계(입력 → 요청 실행 → 응답 표시)

**완료 확인 방법**
- 화면 구성 문서에서 각 패널의 역할/입출력 정의가 존재하는지 확인
- 탭/Environment 구성에 대한 사용자 동선이 명시되었는지 확인

---

### M2: HTTP 실행 + 표시
**작업 항목**
- 요청 모델(메서드, URL, 헤더, 바디, 타임아웃) 설계
- 응답 표시 포맷(Body/Headers/Status/Elapsed) 설계
- Auth 유형(Basic/Bearer) 파라미터 정의

**산출물**
- 요청/응답 데이터 모델 설계 문서
- UI ↔ Core 인터페이스 초안

**완료 확인 방법**
- 요청/응답 데이터 구조에 필요한 필드가 모두 포함되었는지 확인
- Auth 설정이 데이터 모델에 반영되었는지 확인

---

### M3: 시스템 & 네트워크 고급
**작업 항목**
- Proxy/SSL Verification/Redirect 설정 항목 정의
- Logging 및 Global Exception Handler 설계
- Environment 변수 치환 규칙 정의 (`{{var}}`)

**산출물**
- 네트워크 설정 옵션 명세
- 로깅 정책(파일/콘솔, 레벨, 저장 위치) 문서
- 변수 치환 규칙 문서

**완료 확인 방법**
- Proxy/SSL/Redirect 옵션이 문서에 구체적으로 명시되었는지 확인
- 로깅 레벨/포맷/경로가 정의되었는지 확인
- 변수 치환 예시가 1개 이상 포함되었는지 확인

---

### M4: Cancel
**작업 항목**
- 비동기 실행 구조(QThread/QRunnable) 설계
- Cancel 시 UI 상태 복구 흐름 정의

**산출물**
- 비동기 처리 플로우(스레드/시그널) 문서
- 취소/완료/실패 상태 전이 정의

**완료 확인 방법**
- 상태 전이 표에 “취소” 경로가 포함되었는지 확인
- UI 복구 조건(버튼 상태/스피너/상태바)이 명시되었는지 확인

---

### M5: JSON 저장/로드
**작업 항목**
- Workspace JSON 스키마 정의(정규화 구조)
- Atomic write 절차 정의
- Environment 데이터 포함 여부 확인

**산출물**
- JSON 스키마 문서(필수/선택 필드)
- 저장 절차 문서(임시파일 → fsync → rename)

**완료 확인 방법**
- `schema_version`, `collections`, `folders`, `requests`, `environments` 필드가 포함되었는지 확인
- Atomic write 단계가 명확히 기술되었는지 확인

---

### M6: History JSONL
**작업 항목**
- history.jsonl 포맷 설계(요청/응답 최소 필드)
- UI 표시 요구사항 정의(최근 요청 리스트)

**산출물**
- JSONL 기록 포맷 문서
- History UI 요구사항 문서

**완료 확인 방법**
- JSONL 항목에 timestamp, method, url 등이 포함되었는지 확인
- UI 표시 항목 목록이 정의되었는지 확인

---

### M7: Workspace 저장/로드 UI 연동
**작업 항목**
- 저장/로드 메뉴(또는 버튼) UX 정의
- 저장/로드 성공 알림을 상단 배너로 표시하는 정책 정의
- Workspace 모델과 UI 상태 매핑 규칙 정의
- 저장 경로 및 기본 파일명 정책 정의

**산출물**
- 저장/로드 UI 플로우 문서
- 저장/로드 성공 알림 배너 정책 문서
- Workspace ↔ UI 매핑 규칙 문서

**완료 확인 방법**
- 저장/로드 플로우에 필요한 입력/결과가 명시되었는지 확인
- 성공 알림이 대화상자 대신 상단 배너로 표시되는지 확인
- UI 상태가 Workspace 필드에 어떻게 매핑되는지 표로 정리되어 있는지 확인

---

### M8: History UI 구현
**작업 항목**
- History 리스트 패널 레이아웃 설계
- 항목 클릭 시 요청 상세 표시 동작 정의
- 성공/실패 상태 표시 스타일 정의

**산출물**
- History UI 레이아웃 문서
- 항목 선택 동작 시퀀스 문서

**완료 확인 방법**
- 리스트 컬럼(시간/메서드/URL/상태/지연)이 정의되었는지 확인
- 클릭 동작이 요청/응답 뷰어와 어떻게 연결되는지 설명이 있는지 확인

---

### M9: 배포/패키징 준비
**작업 항목**
- PyInstaller 빌드 스펙 정의
- 리소스 파일 포함 규칙 정의
- 버전/릴리즈 노트 규칙 정의

**산출물**
- 빌드 스펙/절차 문서
- 릴리즈 체크리스트

**완료 확인 방법**
- 빌드 명령과 산출물 위치가 명확히 문서화되었는지 확인
- 릴리즈 체크리스트에 버전/변경 로그 항목이 포함되어 있는지 확인
